  2. Describe your design for your site’s shopping cart. Will it be a separate page that the user can view and edit, or will it be integrated into the product pages? If so, describe in detail how this will work on your site. Provide several examples of using the cart.

    3. Before the user is logged in, they can view the products, home, and the about page. The user can add to the cart, and the cart counter will go up. Once done shopping, they can click the cart to go to the shopping cart page. To check, the user must be logged in: if they aren't logged in, it will take them to the login page. Users can remove items and select a higher quantity than the default one. It should limit them to the amount available. Once done editing their cart, the user will click purchase. Then once logged in, they may check out and view their invoice.


    A user may want to buy shoes from multiple brands. The site will allow users to add shoes from three brands. For every shoe added to the cart, the item counter will increase by 1. Then if they want to purchase multiple of one shoe, they can change the quantity on the shopping car page. Say the user decides they want to add another shoe. They may go back to the products page and add more shoes. And if the user decides they no longer want a shoe, they can go back to the shopping cart and use the remove button.

  4. How will you avoid access to your application when the user has not logged in or registered? What are the particular security concerns you must address?

    5. I will use an if statement checking if the current session has the user’s status as a successful login. The status will change on the server; the user can’t manually change the status without a successful login. Users can see their invoice if they pass the “if statement “on the server.

    I must address if the user were to be inactive on my site. Say they were logged in on a public device like, at the library, and their account could be compromised. To combat this, I have the server check if the user has been inactive for more than 10mins. If the user is, then it will log them out.

    Session and cookies will be used to check whether a user has logged in. This way it won’t matter where the user was redirected from, like in the prior assignment. I also must make sure all my passwords are stored with encryption.


  5. Upon successful login, how do you provide personalization in your UI? Explain how you did or will do this (paste code if necessary)

    6. Upon successful login, underneath the nav bar, there will be a display, “Welcome ,” and the status says the user is logged in. There will also be an item counter in the nav bar to show the user how many different kinds of shoes they’ve added. Moreover, their name will be displayed on the invoice, thanking them for their order. The invoice will also display the number of users logged in.
